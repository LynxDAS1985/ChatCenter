## 🎓 Контекст проекта

**Название**: ЦентрЧатов (рус.) / ChatCenter (eng.)

**Что это**: Electron-приложение — единое окно для работы с несколькими мессенджерами одновременно, с встроенным ИИ-помощником для быстрых ответов, шаблонов и автоответа.

**Целевая аудитория**: Операторы и менеджеры, работающие с клиентами через несколько мессенджеров (Telegram, WhatsApp, VK, Viber и др.).

**Текущая версия**: v0.60.8 (12 марта 2026)

---

## 🧩 Основные модули

| Модуль | Кодовое имя | Описание |
|--------|-------------|----------|
| Мессенджеры | `Messengers` | Управление WebView-вкладками мессенджеров (Telegram Web, WhatsApp Web, VK и др.) |
| Монитор чатов | `ChatMonitor` | Перехват и чтение входящих сообщений через инъекцию скриптов в WebView |
| ИИ-помощник | `AIAssistant` | Анализ входящих сообщений, генерация вариантов ответа через LLM |
| Шаблоны ответов | `Templates` | Библиотека готовых текстов и фраз, категории, поиск |
| Авто-ответчик | `AutoReply` | Правила автоответа: по расписанию, по ключевым словам, по чату |
| Расписание | `Scheduler` | Временные окна активности авто-ответчика (рабочие часы, выходные) |
| Настройки | `Settings` | Конфигурация мессенджеров, ИИ-провайдеров, правил, профилей |

## 🏗️ Архитектурные принципы

**Технологии**:
- **Electron** — оболочка приложения
- **WebView** (`<webview>` тег) — изолированный рендер веб-версий мессенджеров
- **IPC** (ipcMain / ipcRenderer) — единственный канал связи main ↔ renderer
- **preload.js** — мост между main и renderer (contextBridge)

**Критически важно**:
- Всё через IPC — никакого `require('electron')` в renderer-процессах
- Инъекция скриптов в WebView только через `preload` WebView или `executeJavaScript`
- Каждый мессенджер — отдельный изолированный `<webview>` с собственной сессией/partition
- Перехват сообщений — через MutationObserver внутри WebView, результат передаётся через IPC
- ИИ-запросы только из main-процесса (API-ключи не в renderer)

## 📌 Обновление версии (ОБЯЗАТЕЛЬНО при каждом изменении кода)

При изменении кода **ОБЯЗАТЕЛЬНО** обновить версию **ВО ВСЕХ местах**:

| # | Файл | Что менять |
|---|------|-----------|
| 1 | `package.json` | `"version": "X.Y.Z"` |
| 2 | `src/main/settings.html` | Версия в заголовке окна настроек |
| 3 | `CLAUDE.md` | `**Текущая версия**: vX.Y.Z (дата)` |
| 4 | `.memory-bank/features.md` | Версия в шапке + запись в Changelog |

### Правила версионирования
- **Patch** (0.1.X → 0.1.X+1): багфиксы, мелкие правки
- **Minor** (0.X.0): новые функции, новый мессенджер, изменения UI
- **Major** (X.0.0): только по согласованию с пользователем

### Чеклист перед сборкой
1. ✅ `package.json` — версия обновлена
2. ✅ `CLAUDE.md` — версия и дата обновлены
3. ✅ `.memory-bank/features.md` — версия + запись в Changelog
4. ✅ Если есть новые файлы — добавлены в `package.json → build.files`
