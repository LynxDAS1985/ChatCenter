### 📦 Архив — `.memory-bank/archive/` (НЕ читать по умолчанию)

В папке `.memory-bank/archive/` лежат **неактуальные** файлы и секции: завершённые handoff'ы, отменённые планы, секции ⚪ ИСТОРИЯ (РЕШЕНО).

**Правила работы с архивом**:
- ❌ Агент **НЕ читает** `archive/` по умолчанию (не включать в обязательные чтения, не лезть туда при отладке)
- ❌ `Grep`/`Glob` по `.memory-bank/` **должны исключать** `archive/` (иначе архив съест контекст впустую)
- ✅ Читать только если пользователь явно попросил: «загляни в архив», «что было в старом handoff», «почему мы решили так»
- ✅ При архивации файла: имя `YYYY-MM-<имя>.md` + запись в журнал [`archive/README.md`](.memory-bank/archive/README.md)

**Что архивировать**:
- Завершённые handoff'ы (2+ недели стабильности в релизе)
- Отменённые/замороженные планы (>3 месяцев бездействия)
- Секции ⚪ ИСТОРИЯ (РЕШЕНО) из активных файлов

**Что НЕ архивировать**:
- Живые файлы, даже если давно не обновлялись
- Раздутые справочники — их **разбивают** на темы (как `common-mistakes.md` → `mistakes/`), а не прячут в архив

**Почему правило «не читать без просьбы»**: архив — это история проекта для разбора разовых вопросов. Его автоматическое чтение при каждой задаче тратит контекст на то, что уже не влияет на код.

## 🎓 Контекст проекта

**Название**: ЦентрЧатов (рус.) / ChatCenter (eng.)

**Что это**: Electron-приложение — единое окно для работы с несколькими мессенджерами одновременно, с встроенным ИИ-помощником для быстрых ответов, шаблонов и автоответа.

**Целевая аудитория**: Операторы и менеджеры, работающие с клиентами через несколько мессенджеров (Telegram, WhatsApp, VK, Viber и др.).

**Текущая версия**: v0.87.56 (24 апреля 2026)

---

## 🧩 Основные модули

| Модуль | Кодовое имя | Описание |
|--------|-------------|----------|
| Мессенджеры | `Messengers` | Управление WebView-вкладками мессенджеров (Telegram Web, WhatsApp Web, VK и др.) |
| Монитор чатов | `ChatMonitor` | Перехват и чтение входящих сообщений через инъекцию скриптов в WebView |
| ИИ-помощник | `AIAssistant` | Анализ входящих сообщений, генерация вариантов ответа через LLM |
| Шаблоны ответов | `Templates` | Библиотека готовых текстов и фраз, категории, поиск |
| Авто-ответчик | `AutoReply` | Правила автоответа: по расписанию, по ключевым словам, по чату |
| Расписание | `Scheduler` | Временные окна активности авто-ответчика (рабочие часы, выходные) |
| Настройки | `Settings` | Конфигурация мессенджеров, ИИ-провайдеров, правил, профилей |

## 🏗️ Архитектурные принципы

**Технологии**:
- **Electron** — оболочка приложения
- **WebView** (`<webview>` тег) — изолированный рендер веб-версий мессенджеров
- **IPC** (ipcMain / ipcRenderer) — единственный канал связи main ↔ renderer
- **preload.js** — мост между main и renderer (contextBridge)

**Критически важно**:
- Всё через IPC — никакого `require('electron')` в renderer-процессах
- Инъекция скриптов в WebView только через `preload` WebView или `executeJavaScript`
- Каждый мессенджер — отдельный изолированный `<webview>` с собственной сессией/partition
- Перехват сообщений — через MutationObserver внутри WebView, результат передаётся через IPC
- ИИ-запросы только из main-процесса (API-ключи не в renderer)

## 🚫 Лимиты размера файлов (ОБЯЗАТЕЛЬНО соблюдать)

Проект защищён от бесконтрольного роста файлов. Тесты `npm test` автоматически проверяют лимиты.

### Лимиты на файлы (v0.85.0+)

| Файл | Лимит | Текущий | Если превышен |
|------|-------|---------|--------------|
| Любой `.jsx` компонент | **600 строк** | — | Разбить на подкомпоненты или вынести логику в utils |
| Любой `.js` утилита | **300 строк** | — | Разбить на несколько файлов по функциям |
| Любой React hook | **130 строк** | — | Разбить на несколько хуков |
| `src/App.jsx` | **600 строк** | 556 | Вынести в `src/hooks/` или `src/components/` |
| `main/main.js` | **600 строк** | 569 | Вынести в `main/handlers/` или `main/utils/` |
| `monitor.preload.cjs` | **600 строк** | 465 | Вынести в `main/preloads/utils/` |
| `webviewSetup.js` | **600 строк** | 545 | Вынести в `src/utils/` |
| `AISidebar.jsx` | **600 строк** | 542 | Вынести в `src/utils/ai*.js` |
| `dockPinHandlers.js` | **600 строк** | 571 | Вынести в `main/handlers/dockPinUtils.js` |

**Тест `fileSizeLimits.test.cjs` автоматически проверяет все лимиты.** Если файл перерос — тест падает.

### Лимиты на функции

| Правило | Лимит | Если превышен |
|---------|-------|--------------|
| Одна функция | **100 строк** | Разбить на подфункции |
| Inline `executeJavaScript` | **50 строк** | Вынести скрипт в `messengerConfigs.js` |
| React useEffect | **30 строк** | Вынести логику в отдельную функцию |

### Правила добавления нового кода

1. **Новая бизнес-логика** → `src/utils/имяМодуля.js` + тест в `src/__tests__/`
2. **Новый UI-блок** → `src/components/ИмяКомпонента.jsx` (если > 50 строк)
3. **Новый IPC handler** → добавить в `main/main.js` + тест в `ipcChannels.test.js`
4. **Новый спам-паттерн** → `shared/spamPatterns.json` (единый источник)
5. **Новый мессенджер** → конфиг в `src/utils/messengerConfigs.js`

### Как проверить

```bash
npm test  # 410+ тестов, включая проверку лимитов размеров файлов
```

## 📌 Обновление версии (ОБЯЗАТЕЛЬНО при каждом изменении кода)

При изменении кода **ОБЯЗАТЕЛЬНО** обновить версию **ВО ВСЕХ 4 местах**:

| # | Файл | Что менять |
|---|------|-----------|
| 1 | `package.json` | `"version": "X.Y.Z"` |
| 2 | `package-lock.json` | корневой `"version"` + `packages[""].version` |
| 3 | `CLAUDE.md` | `**Текущая версия**: vX.Y.Z (дата)` — в начале **и** в подвале файла |
| 4 | `.memory-bank/features.md` | Версия в шапке + запись в Changelog |

### Правила версионирования
- **Patch** (0.1.X → 0.1.X+1): багфиксы, мелкие правки, рефакторинг памяти
- **Minor** (0.X.0): новые функции, новый мессенджер, изменения UI
- **Major** (X.0.0): только по согласованию с пользователем

### Чеклист перед сборкой
1. ✅ `package.json` — версия обновлена
2. ✅ `package-lock.json` — **две** `"version"`-записи обновлены
3. ✅ `CLAUDE.md` — версия и дата в двух местах
4. ✅ `.memory-bank/features.md` — версия + запись в Changelog
5. ✅ Если есть новые файлы — добавлены в `package.json → build.files`
