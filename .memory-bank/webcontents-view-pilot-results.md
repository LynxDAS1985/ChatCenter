## Известные ограничения пилота

1. **ChatMonitor НЕ интегрирован** — это требует Phase 2.3 (full) — переписки `webviewSetup.js` (~400 строк)
2. **`<webview>` тег НЕ удалён** — это fallback при flag=OFF. Удаление только после полной миграции
3. **executeJavaScript / send** работают через wcv:* IPC, но Slot не имеет прокси-API — нужен webContentsViewBridge.js (план)

## Как включить пилот (для тестера)

1. Запустить приложение
2. ⚙️ Настройки → найти «WebContentsView (экспериментально, требует перезапуска)»
3. Включить toggle → закрыть Настройки
4. **Полностью закрыть приложение** (выйти из трея)
5. Запустить снова
6. Открыть любой мессенджер (НЕ ChatCenter native)
7. Заполнить чек-лист выше, добавить в этот файл новую запись пилота

## Как откатить пилот

1. ⚙️ Настройки → выключить toggle
2. Перезапустить приложение
3. Старый `<webview>` тег снова активен — ChatMonitor работает

## Критерии для Phase 2.3 (full)

Запускаем полную миграцию `webviewSetup.js` только если:

| Критерий | Минимум для запуска |
|---|---|
| Пилотов проведено | ≥ 3 разными мессенджерами |
| Стабильность | 0 крашей за 30 минут активной работы |
| Производительность | Память ≤ baseline (`<webview>`) ± 20% |
| Разделитель AI sidebar | 100% не залипает в пилоте |
| Память за 1 час | Не растёт более чем на 100 МБ при активной работе |
| Готовность переписать `webviewSetup.js` | 2-3 дня свободного времени + 1 неделя на стабилизацию |
