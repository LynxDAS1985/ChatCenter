## 🔴 Разбиения которые я НЕ сделал (и ты бы сделал)

### 1. `main/native/telegramHandler.js` — 1260 строк (!)

**Почему не сделал**: пользователь явно сказал «файлы не разбивай, просто чтобы другой ИИ увидел». Поэтому я зафиксировал в тесте как **исключение с потолком 1300** — сейчас проходит, но при 1300+ упадёт.

**Как разбить** (рекомендация):

```
main/native/
├── telegramHandler.js        [оставить тонкий роутер ~100 строк]
├── telegramAuth.js           [вход, сессии, 2FA ~300 строк]
├── telegramMessages.js       [отправка/приём сообщений ~400 строк]
├── telegramChats.js          [список чатов, unread, markRead ~300 строк]
└── telegramMedia.js          [фото, видео, файлы, stickers ~260 строк]
```

**Что там внутри** (ориентируйся):
- `tg:init`, `tg:login` → в `telegramAuth.js`
- `tg:send-message`, `tg:new-message listener`, `tg:edit-message` → в `telegramMessages.js`
- `tg:get-chats`, `tg:chat-unread-sync`, `tg:mark-read` → в `telegramChats.js`
- `tg:get-media`, `tg:download-media` → в `telegramMedia.js`

**Риск**: внутри есть общий state (клиент GramJS, кэш). Нужно вынести его в отдельный модуль `telegramClient.js` и импортировать во все 4 файла.

### 2. `src/native/modes/InboxMode.jsx` — 765 строк

**Почему не сделал**: та же причина — пользователь запретил разбивать.

**Как разбить**:

```
src/native/modes/
├── InboxMode.jsx             [главный контейнер ~300 строк]
├── InboxMessageList.jsx      [список сообщений + scroll ~250 строк]
└── InboxHeader.jsx           [шапка чата + поиск ~150 строк]
```

**Что там внутри** (ориентируйся):
- Сам InboxMode — связка store + activeChat + useEffect'ы → оставить 300 строк
- Рендер списка сообщений (`visibleMessages.map(...)`) + scroll logic → вынести в `InboxMessageList`
- Header с именем чата, аватаркой, кнопками → вынести в `InboxHeader`

**Риск**: `seenChatsRef` (v0.87.67) и `initialScrollDoneRef` используются в двух местах — нужна аккуратная передача через props.
