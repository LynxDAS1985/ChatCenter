# Архив `.memory-bank/archive/`

Здесь лежат **неактуальные** файлы и секции из Memory Bank.

## ⚠️ Правило для AI

**Агент НЕ читает файлы из этой папки по умолчанию.**

Читать только если:
- пользователь явно попросил («загляни в архив», «что было в старом handoff»)
- текущая задача — «восстановить удалённое», «разобраться почему решили так-то»

Поиск (`Grep`, `Glob`) по `.memory-bank/` **должен исключать** `archive/` — иначе архив будет съедать контекст впустую.

## Что сюда попадает

| Тип | Пример | Когда архивировать |
|---|---|---|
| Завершённые handoff-документы | `native-scroll-diagnostics-handoff.md` | Когда баг закрыт в релизе и 2+ недели подтверждён пользователем |
| Отменённые планы и ADR | `native-mode-plan.md` | Если решение отменено или заморожено > 3 месяцев |
| Секции ⚪ ИСТОРИЯ из активных файлов | Блоки «РЕШЕНО в vX.Y.Z» | Когда корневая причина устранена и запись сохраняется только как учебный пример |
| Исторические changelog'и | Старый `features.md` когда > 200 КБ | Когда `features.md` становится неподъёмным для чтения |

## Что сюда НЕ попадает

- ❌ **Активные файлы** — даже если давно не обновлялись. Если ИИ может понадобиться на них посмотреть — они живые.
- ❌ **Раздутые живые справочники** — их нужно **разбивать** (см. `.memory-bank/mistakes/`), а не прятать в архив.
- ❌ **Файлы в работе** — даже если план частично реализован.

## Правила именования

`YYYY-MM-<имя>.md` — дата вывода в архив в имени.

Примеры:
- ✅ `2026-04-common-mistakes-resolved.md`
- ✅ `2026-06-native-scroll-handoff.md`
- ❌ `old-handoff.md` (нет даты)
- ❌ `common-mistakes-archive.md` (нет даты)

## Как восстановить из архива

Если файл снова стал актуален (например, баг вернулся):
1. Скопировать нужный блок обратно в активный файл (не перемещать — пусть живёт в обоих местах временно)
2. В журнале выше пометить: «Частично восстановлено DD.MM.YYYY — см. такой-то файл»
3. Через 2 недели, если всё стабильно — удалить из архива или оставить как есть
