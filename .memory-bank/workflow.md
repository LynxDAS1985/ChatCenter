# Рабочий процесс — ChatCenter

**Версия**: v0.87.62 (24 апреля 2026)
**Главный документ**: [`../CLAUDE.md`](../CLAUDE.md) — правила поведения AI.
**Этот файл** — рабочий процесс и чеклисты задач. Дополняет CLAUDE.md, не заменяет.

---

## 🧠 Перед каждой задачей

1. Прочитать [`../CLAUDE.md`](../CLAUDE.md)
2. Прочитать этот файл + [`README.md`](./README.md) — карта памяти
3. Прочитать **только нужные** файлы по теме:
   - **Постоянные** (всегда): `architecture.md`, `coding-rules.md`, `workflow.md`
   - **По теме**: `api.md` / `messengers.md` / `ai-integration.md` / `autoreply.md` / `ui-components.md` / `decisions.md`
   - **Баг/регрессия**: `common-mistakes.md` (индекс) → **только нужный** файл из `mistakes/`
   - **Узкие** (только если задача точно про них): `native-scroll-diagnostics-handoff.md`, `native-mode-plan.md`
4. **НЕ** читать архив (`archive/`) без явной просьбы пользователя
5. Сформулировать **критерий готовности**: «Задача готова, когда: …»
6. Прочитать код в области задачи — **не угадывать**, читать

## ⚡ Во время работы

- **Surgical changes**: менять только связанные с задачей файлы, не рефакторить соседний код
- **Simplicity first**: минимально достаточное решение, без «на будущее»
- **Параллельность**: читать файлы одновременно, делать независимые правки одновременно
- **НЕ параллелить**: git-команды, `npm install`, build, e2e, запись в один файл
- Один логический запрос = один логический commit
- После значимого изменения — обновить нужный файл Memory Bank

### ⚠️ Запреты (без явного разрешения пользователя)

- `npm start`, `npm run dev`, `npm test`, e2e — запускают приложение
- `npm install`, `npm ci` — перезаписывает git hooks
- `git reset`, `git checkout`, `git revert`, `--no-verify`, `git add -A`
- Переименование/удаление файлов, изменения структуры проекта
- Полный список и причины — в [`../CLAUDE.md`](../CLAUDE.md) → «🛑 КРИТИЧЕСКИЕ ЗАПРЕТЫ»

### ✅ Безопасные проверки (без подтверждения)

- `npm run lint` — ESLint
- `npm run test:vitest` — Vitest в jsdom (без Electron)
- `npm run check-memory` — здоровье Memory Bank
- `npm run regen-claude-structure` — регенерация таблицы памяти в CLAUDE.md
- `node src/__tests__/<имя>.test.cjs` — точечный тест
- `git status`, `git diff`, `git log`

### 🔄 Конфликт память vs код

Если `.memory-bank/*` или auto-memory говорит одно, а реальный код — другое → **доверяй коду**. Обнови запись в памяти. Если не уверен — спроси пользователя.

## 📝 После задачи

1. Обновить версию во всех 4 местах (см. CLAUDE.md → «Обновление версии»):
   - `package.json`
   - `package-lock.json` (две `version`-записи)
   - `CLAUDE.md` (в начале и в подвале)
   - `.memory-bank/features.md` (в шапке + запись в Changelog)
2. Обновить `.memory-bank/`:
   - Новые ловушки → в нужный файл `mistakes/<тема>.md` (**не** в `common-mistakes.md` — он индекс)
   - Новые решения → `decisions.md`
   - Новая фича/фикс → `features.md` (Changelog)
   - Структурные изменения памяти → `CHANGELOG.md`
3. Проверки (по риску):
   - `npm run lint` + `npm run test:vitest` для большинства задач
   - Отдельные unit-тесты по области изменения
   - Новая логика без тестов → создай минимальный тест
4. Commit + push — **только если пользователь явно попросил**
5. Попросить пользователя **запустить и проверить** (UI-изменения — визуально)

## 📋 Pipeline по размеру задачи

### Простая (1 файл, 1 правка)
1 ход — прочитал → изменил → проверил → отчитался.

### Средняя (2–5 файлов)
- **ХОД 1**: параллельные чтения + критерий готовности
- **ХОД 2**: все правки + версия в 4 местах
- **ХОД 3**: обновить Memory Bank + проверки + commit/push (если просили)
- **ХОД 4**: итог + 5 советов + таблица «Что сделано»

### Крупная (>5 файлов / новая фича / архитектурный риск)
Сначала **план с числом ходов** → потом выполнение.

После каждого хода: `ХОД X/N: ✅ [что сделано]`

## 🆘 Если что-то не работает

Краткая справка. Полная таблица — в [`../CLAUDE.md`](../CLAUDE.md) → «🆘 ЕСЛИ ЧТО-ТО НЕ РАБОТАЕТ».

| Ситуация | Что делать |
|---|---|
| Pre-commit hook падает | `bash scripts/check-memory.sh` — покажет причину |
| Версии разошлись | `npm run check-memory` → выровнять 4 места |
| Тест `featuresReferences` ругается | Битая ссылка в последних 10 версиях features.md — удали упоминание или восстанови файл |
| Тест `memoryBankSizeLimits` ругается | Файл памяти перерос лимит — разбить (см. примеры в CHANGELOG.md) |
| Контекст переполнен | Читай только индекс `common-mistakes.md` + нужную тему `mistakes/` |
| Таблица «Структура памяти» в CLAUDE.md устарела | `npm run regen-claude-structure` |
