# Рабочий процесс — ChatCenter

## Правила работы AI-разработчика

### Перед каждой задачей

1. Прочитать `CLAUDE.md`
2. Прочитать `.memory-bank/README.md`
3. Прочитать файлы Memory Bank по теме задачи
4. Прочитать существующий код в области задачи (не угадывать, читать!)
5. Составить план действий в голове (не писать код сразу)

### Во время работы

- Менять только то, что нужно для задачи — не рефакторить попутно
- Одна задача — один коммит (логически)
- После каждого значимого изменения — обновить нужный файл Memory Bank
- Если что-то непонятно — спросить пользователя, не угадывать

### После задачи

- Обновить версию во всех 4 местах (см. CLAUDE.md)
- Обновить `features.md` — добавить в Changelog
- Обновить нужные файлы Memory Bank
- Попросить пользователя запустить и проверить

## Планирование

При получении новой задачи:

1. **Понять** — что именно нужно сделать (переспросить если неясно)
2. **Найти** — где в коде это будет (прочитать существующий код)
3. **Спланировать** — список файлов для изменения
4. **Сделать** — минимально необходимые изменения
5. **Обновить** — Memory Bank + версию
6. **Попросить проверить** — пользователь запускает и тестирует

## Запрещено без спроса

- Переименовывать файлы и папки
- Менять структуру проекта
- Удалять код/файлы
- Устанавливать новые зависимости
- Изменять `package.json` (кроме версии)
- Делать рефакторинг не связанный с задачей
