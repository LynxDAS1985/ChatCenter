## Компонент: Вкладка мессенджера (MessengerTab)

Расположение: горизонтальная строка в шапке (h-[48px])
Размеры: `h-[40px] px-3` — бейдж и крестик в flex-потоке (`ml-auto shrink-0`), вкладка расширяется автоматически (v0.19.6)

### Состояния вкладки

**Активная вкладка:**
- `backgroundColor: ${color}1A` — 10% прозрачность бренд-цвета
- `borderBottom: 2px solid ${color}` — нижняя линия бренд-цвета
- `borderRadius: 6px 6px 0 0`
- Индикатор-точка: `backgroundColor: ${color}` — полный цвет
- Название: `color: ${color}` — бренд-цвет

**Неактивная вкладка:**
- Фон: `transparent`
- `borderBottom: 2px solid transparent`
- Hover: `backgroundColor: rgba(255,255,255,0.06)`
- Индикатор-точка: `backgroundColor: ${color}55` — 33% прозрачность
- Название: `color: rgba(255,255,255,0.45)` — приглушённый белый

### Структура вкладки (JSX)

```jsx
<button style={{ backgroundColor, borderBottom, borderRadius }}>
  {/* Цветная точка-индикатор */}
  <span style={{ backgroundColor: isActive ? color : `${color}55` }} />

  {/* Название + зум + аккаунт (центрировано) */}
  <span className="flex flex-col items-center leading-tight">
    <span style={{ color: isActive ? color : 'rgba(255,255,255,0.45)' }}>
      {name}
    </span>
    {accountInfo && (
      <span className="text-[10px] max-w-[110px] overflow-hidden text-ellipsis"
            style={{ color: isActive ? `${color}AA` : 'rgba(255,255,255,0.25)' }}>
        {accountInfo}
      </span>
    )}
  </span>
</button>
```

## Извлечение информации об аккаунте

Реализовано в `src/App.jsx` через `webview.executeJavaScript()`.

**Алгоритм:**
1. WebView вешает слушатель `dom-ready`
2. После `dom-ready` ждём **4 секунды** (страница продолжает загружаться асинхронно)
3. Выполняем `executeJavaScript(accountScript)` — обходим список CSS-селекторов
4. Если результат не найден — повтор через 4 сек., максимум 10 попыток (~40 сек.)
5. Длинные строки (>100 символов) игнорируются (защита от мусора)

**Скрипты по мессенджерам (`accountScript`):**

| Мессенджер | Пробуемые селекторы |
|-----------|---------------------|
| Telegram | **IndexedDB cursor (v0.19.2)**: без DOM! Cursor по `users` store → `pFlags.self === true` → `first_name + last_name` / `+phone` fallback |
| WhatsApp | `[data-testid="profile-details-header-name"]`, `[data-testid="user-preferred-name"]` |
| ВКонтакте | `.TopNavBtn__title`, `.header__top--uname`, `.vkuiSimpleCell__content .vkuiTypography--weight-1` |

⚠️ **Важно**: Селекторы могут устареть при обновлении мессенджеров. Нужно периодически проверять.
⚠️ **Telegram**: НЕ использовать DOM-селекторы для имени аккаунта — все они возвращают имя открытого чата. Использовать только IndexedDB-подход (v0.19.0).

## Компонент: Строка статистики (StatusBar)

Расположение: самый низ приложения, `h-[26px]`, `text-[11px]`, `select-none`.

```
┌──────────────────────────────────────────────────────────────────────────┐
│ 💬 12 сегодня · ⚡ 0 авто · 📊 144 всего · 📥 3 непрочитано    − 100% + │
└──────────────────────────────────────────────────────────────────────────┘
```

- Левая часть: статистика сообщений с `title` тултипами на hover
- Правая часть (`ml-auto`): контролы зума активной вкладки

### Зум-контролы (v0.18.0)

| Элемент | Описание |
|---------|----------|
| `−` кнопка | Уменьшить на 5%, disabled при ≤25% |
| `N%` / input | Клик → поле ввода, Enter/Blur применяет, Escape отменяет |
| `+` кнопка | Увеличить на 5%, disabled при ≥200% |
| `↺` кнопка | Сброс к 100%, появляется только при zoom ≠ 100% |

Диапазон: 25–200%. Шаг: 5%. Каждая вкладка хранит свой зум.
Зум сохраняется в settings (`zoomLevels: { [id]: number }`), загружается при старте.
Применяется через `webview.setZoomFactor(pct / 100)`.

### Цветовая индикация статуса мониторинга (v0.23.0)

Точка-индикатор на вкладке отражает статус мониторинга:
- 🟢 Зелёная (цвет мессенджера) — мониторинг работает (`monitorStatus = 'active'`)
- 🟡 Жёлтая `#eab308` — загрузка, первые 20 сек (`monitorStatus = 'loading'`)
- 🔴 Красная `#ef4444` — монитор не ответил за 20 сек (`monitorStatus = 'error'`)

### Контекстное меню вкладки (v0.23.0)

ПКМ на вкладке мессенджера → выпадающее меню:
| Пункт | Действие |
|-------|----------|
| 🔄 Перезагрузить | `webview.reload()` + сброс monitorStatus в 'loading' |
| 🔍 Диагностика DOM | Повторный вызов `runDiagnostics()` через executeJavaScript |
| 📋 Копировать URL | URL мессенджера в буфер обмена |
| ✕ Закрыть вкладку | Удаление мессенджера |

### Последнее сообщение в статусбаре (v0.23.0)

Вместо диагностики — `💬 Telegram: текст сообщения...` (исчезает через 8 сек).
Диагностика по-прежнему доступна через console.log и контекстное меню → Диагностика DOM.

### Зум из WebView (v0.19.1)
Ctrl+колёсико и Ctrl+клавиши обрабатываются в `monitor.preload.js` → IPC `zoom-change`/`zoom-reset` → App.jsx.
Причина: WebView захватывает события мыши/клавиатуры, renderer не получает их.

### Плавная анимация зума (v0.19.1)
`animateZoom(id, from, to)` — 6 кадров, ease-out квадратичный, `requestAnimationFrame`.

### Индикатор зума на вкладке (v0.19.3)
Бейдж `{zoomLevel}%` рядом с названием мессенджера. Показывается когда zoom ≠ 100%.
Стиль: `text-[9px] font-bold`, цвет = цвет мессенджера, фон = `${color}20`.

### Центрирование элементов вкладки (v0.19.3)
Название, зум-бейдж и подпись аккаунта — всё центрировано по оси (`items-center`) внутри flex-колонки.

## Правила UI

1. Использовать **Tailwind CSS** для статичных классов, **inline styles** только для динамических цветов
2. Избегать хардкода цветов в Tailwind (невозможно использовать `bg-[${dynamicColor}]` в runtime)
3. Hover-эффекты на кнопках — через `onMouseEnter`/`onMouseLeave` для inline-стилей ИЛИ Tailwind `hover:` для статичных
4. Scroll-контейнеры должны быть явно ограничены `overflow-hidden` / `overflow-x-auto`
5. Текст в шапке: `select-none` — нельзя выделять
6. Кнопки в drag-зоне: обязательно `style={{ WebkitAppRegion: 'no-drag' }}`
